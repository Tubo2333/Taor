### [x] P1-5. AnthropicAdapter think() 加重试

**问题**：网络抖动 / 429 限流 → 直接抛异常，无重试。

**修复**：

修改 `packages/adapters/src/anthropic.ts`，在 `think()` 方法里加 retry wrapper。

不需要改 `think()` 本身——加一个私有方法 `withRetry<T>(fn: () => Promise<T>, maxRetries = 4): Promise<T>`：

```typescript
private async withRetry<T>(fn: () => Promise<T>, maxRetries = 4): Promise<T> {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn()
    } catch (err: any) {
      if (attempt >= maxRetries) throw err
      const status = err?.status ?? err?.response?.status
      // Retry on 429 (rate limit), 5xx (server error), network errors
      if (status === 429 || (status && status >= 500) || err?.code === "ENOTFOUND" || err?.code === "ECONNRESET") {
        const delay = Math.min(1000 * Math.pow(2, attempt), 16000)
        console.warn(`[AnthropicAdapter] Retry ${attempt + 1}/${maxRetries} after ${delay}ms (status=${status ?? err.code})`)
        await new Promise(r => setTimeout(r, delay))
        continue
      }
      throw err // 4xx other than 429 → no retry
    }
  }
  throw new Error("Unreachable")
}
```

然后在 think() 的 AsyncGenerator 包装层：stream 消费在 `withRetry` 外面做，retry 重试整个 `buildRequest + think 启动`（因为 streaming 中断后不能续传）。

实际做法：把 `think()` 里的 HTTP 调用部分包进 `withRetry`。

**改完后验证**：需要能 mock Anthropic SDK。TG2 不做完整测试，但代码逻辑 review 确认 retry 逻辑正确。
